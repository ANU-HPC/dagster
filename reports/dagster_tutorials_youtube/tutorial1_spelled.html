Satisfiability problem - SAT

What is a SAT problem?

  • A set of binary variables: $ x_1,x_2,\dots,x_n $
  • A set of constraints those variables should satisfy
Represented as Conjunctive Normal Form (CNF), conjunction (AND) over disjunctions (OR), over variables or their negation (literal). $$\left(x_1 \vee \neg x_2\right) \wedge \left(\neg x_2 \vee x_3\right)$$ 

						p cnf 3 2
						1 -2 0
						-2 3 0

Example: Sudoku

Example: Sudoku

Boolean $x_{i,j,v}$ iff number $v$ in square $i,j$
  • If a number in a square, any other number is not: $$\wedge_{i,j,v'\ne v}\left( \neg x_{i,j,v} \vee \neg x_{i,j,v'} \right) $$
  • There is atleast one number in a square $$\wedge_{i,j}\left( \vee_{v}x_{i,j,v} \right) $$

Example: Sudoku

  • Each row and column has every number: $$\wedge_{v,j}\left( \vee_{i} x_{i,j,v} \right)~~~~~~\wedge_{v,i}\left( \vee_{j} x_{i,j,v} \right)$$
  • Each box has every number: $$\wedge_{[i,j]\in\{0,1,2\},v} \left( \vee_{[n,m]\in\{0,1,2\}}x_{3i+n,3j+m,v} \right)$$

Example: Sudoku

Python code to generate Sudoku Puzzle:

D = 3 # Subgrid dimension
N = D*D # Grid dimension
cls = []  # The clauses

#variable index for r,c,v
var = lambda r, c, v: (r-1)*N*N+(c-1)*N+v

for r in range(1,N+1):
	for c in range(1, N+1):
		# The cell at (r,c) has at least one value
		cls.append([var(r,c,v) for v in range(1,N+1)])
		# The cell at (r,c) has at most one value
		for v in range(1, N+1):
			for w in range(v+1,N+1):
				cls.append([-var(r,c,v), -var(r,c,w)])

for v in range(1, N+1): # each value v
	# Each row has the value v
	for r in range(1, N+1):
		cls.append([var(r,c,v) for c in range(1,N+1)])
	# Each column has the value v
	for c in range(1, N+1):
		cls.append([var(r,c,v) for r in range(1,N+1)])
	# Each subgrid has the value v
	for sr in range(0,D):
		for sc in range(0,D):
			cls.append([var(sr*D+rd,sc*D+cd,v)
			  for rd in range(1,D+1) for cd in range(1,D+1)])

# TODO: Add clues constraints
cls.append([var(1,1,5)])
# .......

# Output the DIMACS CNF representation
print("p cnf %d %d" % (N*N*N, len(cls)))
for c in cls:
	print(" ".join([str(l) for l in c])+" 0")

Backtracking Search solving process

features of Backtracking Search

  • Systematic and complete: will find a solution or return UNSAT
  • Different variable selection Heuristics
  • Adaptable: clause learning and restarts

Implications

  • Clause satisfied - woohoo
  • Contradiction - triggering a backtrack
  • Clause unsatisfied, by >1 variable
  • Clause unsatisfied, by one variable

The most potent situation is the last, where -4 is forced.

  • Implication chains leading to contradiction occur multiple ways
  • Solution: Add logically redundant clauses to block the implication chain

CDCL solving

Learnt clauses is characteristic of
'Conflict Driven Clause Learning' solvers.
  • Multiple ways of generating different cuts (1UIP is common)
  • Different variable selection rules - eg. VSIDS prioritises selection of variables in learnt clauses
  • Different clause forgetting processes - eg. based on clause activity
  • Restart policies
loads of flexibility, embodied in different solvers (eg. Minisat, Lingeling, Tinisat, etc, etc)

Intermission: Summary

  • SAT problems are binary constraint satisfaction problems, constraints in CNF, defined by OR of litterals
  • backtracking search (DPLL,CDCL) systematically scans through space of binary decisions
  • decisions lead to implication chains, which can be broken by adding learnt clauses
  • learnt clauses accelerate the solving process

Stochastic Local Search

An entirely different process, by assigning all binary variables a value and flipping to increase the number of satisfied clauses.

Modifications to SLS process

Ways to escape local maxima:
  • Add random step
  • Integrate a full random restart
  • Integrate a tabu - self avoiding walk
  • Add dynamic clause weighting
  • etc.

Differences between CDCL and SLS

  • only CDCL can generate UNSAT
  • only CDCL can guarantee finishing
  • SLS and CDCL adept at different classes of problems

Conclusion: Summary

  • SAT problems are binary constraint satisfaction problems, constraints in CNF, defined by OR of litterals
  • backtracking search (DPLL,CDCL) systematically scans through space of binary decisions
  • decisions leads to implication chains, which can be broken by adding learnt clauses
  • learnt clauses build on each other accelerate solving process
  • SLS is non systematic search, which leads to advantages on different problems
